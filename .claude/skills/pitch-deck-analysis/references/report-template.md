# Report Template

Render the analysis in this structure. Lead with the verdict so a busy reader
gets the answer first.

```markdown
# Pitch Deck Analysis — {Company}

**Stage:** {pre-seed / seed / Series A / unknown}  ·  **Slides:** {n}  ·  **Reviewed:** {date}

## Verdict
{2–3 sentences: the headline read. Is this a fundable story today? What's the
single biggest strength and the single biggest gap?}

**Overall: {X}/5** — {one-line rationale}

## Scores
| Dimension | Score | Why |
| --- | --- | --- |
| Problem & market | x/5 | … |
| Solution & product | x/5 | … |
| Traction & validation | x/5 | … |
| Business model & unit economics | x/5 | … |
| Go-to-market | x/5 | … |
| Team | x/5 | … |
| The ask & financials | x/5 | … |

## Coverage
{Table or list: which canonical slides are Present / Weak / Missing.}

## Strengths
- {Specific, cited to a slide.}

## Weaknesses & gaps
- {Specific, cited to a slide, each paired with what would fix it. Mark whether
  it's a presentation problem or a business problem.}

## Red flags
- {Anything deal-stalling. Empty is a fine answer — say "none material."}

## Diligence questions (first-meeting)
1. …
2. …
{5–10 questions an investor would actually ask.}

## Recommendation
{For a founder: the 3 highest-leverage edits before sending this out.
For an investor: pass / explore / lean in, and the one thing to verify.}
```

Keep it tight. The diligence questions and the prioritized fixes are the parts
founders act on — make those the strongest.
